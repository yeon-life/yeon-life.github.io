Chapter 1

도메인이란 무엇인가요?

인터넷 주소, 즉 URL은 숫자로 이루어진 IP 주소(예: 185.199.108.153) 대신 사람이 기억하기 쉬운 이름으로 홈페이지를 찾아갈 수 있게 해주는 인터넷의 이름표입니다.

예시로 이해하기
현재 y-life.kr 빌더로 만든 페이지는 yeon-life.github.io/mypage.html 처럼 GitHub 주소입니다. 도메인을 연결하면 myname.kr 처럼 내 이름이 담긴 짧고 기억하기 쉬운 주소로 바꿀 수 있습니다.
구분도메인 연결 전도메인 연결 후
주소 형태yeon-life.github.io/page.htmlmyname.kr
기억하기 쉬움✗ 어렵다✓ 쉽다
신뢰감✗ 개인 느낌✓ 전문적
SEO (검색 노출)✗ 불리✓ 유리
비용무료.kr 연 약 3,000~5,000원
Chapter 2

도메인 구매 전 확인 사항

  • 원하는 도메인 이름 후보 2~3개 미리 정해두기 (예: myacademy.kr, jinwooschool.com)
  • 가비아(gabia.com) 회원가입 완료
  • 결제 수단 준비 (신용카드 또는 카카오페이)
  • GitHub 계정 로그인 상태 확인 (Pages 설정용)
도메인 이름 선택 팁
짧을수록 좋고, 한글이나 특수문자보다 영문과 숫자 조합이 범용적입니다. .kr 은 한국용, .com 은 글로벌 표준입니다.
Chapter 3

가비아에서 도메인 구매하기

가비아(gabia.com)는 한국에서 가장 많이 사용되는 도메인·호스팅 서비스입니다. .kr 도메인을 연간 약 3,000~5,000원에 등록할 수 있습니다.

1도메인 검색 및 선택

1

가비아 접속

gabia.com에 접속 후 로그인합니다.

2

도메인 검색

메인 페이지 검색창에 원하는 이름을 입력합니다. 예: myacademy

3

사용 가능한 도메인 확인

검색 결과에서 사용 가능 표시된 도메인을 선택합니다. 이미 사용 중이면 다른 이름이나 .net, .co.kr 등 다른 확장자를 시도하세요.

4

장바구니 추가 → 결제

선택한 도메인을 장바구니에 담고 1년 또는 2년 단위로 결제합니다.

결제 완료 후 — 보통 수 분~수 시간 내에 도메인이 활성화됩니다. 가비아 마이페이지에서 확인하세요.
Chapter 4

가비아 DNS 레코드 설정

도메인을 구매했으면, 이제 그 도메인이 내 GitHub Pages 사이트를 가리키도록 DNS 레코드를 설정합니다.

가비아 로그인› 마이가비아› 도메인 관리› 내 도메인› 해당 도메인 클릭› DNS 정보› DNS 레코드 설정

ACNAME 레코드 추가 (www 주소용)

www.내도메인.kr 처럼 www가 붙은 주소에 접속할 때 GitHub Pages로 연결해 줍니다.

타입호스트값 / 주소TTL
CNAME www yeon-life.github.io 600
호스트 칸에 www를 입력하면 됩니다. @ 또는 빈 칸이 아닙니다.

BA 레코드 추가 (루트 도메인용)

www 없이 내도메인.kr 만으로 접속할 때를 위한 설정입니다. A 레코드 4개를 모두 추가합니다.

타입호스트IP 주소TTL
A@185.199.108.153600
A@185.199.109.153600
A@185.199.110.153600
A@185.199.111.153600
호스트를 @ 로 입력하세요 — @ 는 루트 도메인(도메인 자체)을 의미합니다. 가비아에서 빈 칸으로 두면 자동으로 @ 처리되는 경우도 있습니다.

(질문이 쌓이면 실제 스크린샷이 추가됩니다)
DNS 전파 시간 — 레코드를 저장한 후 최대 24~48시간이 소요될 수 있습니다. 보통 수 분~수 시간 내에 완료됩니다.
Chapter 5

GitHub Pages에 커스텀 도메인 등록

DNS 레코드 설정 후, GitHub Pages 설정에서도 내 도메인을 알려줘야 합니다.

1

GitHub 리포지토리 접속

github.com/yeon-life/yeon-life.github.io 에 접속합니다.

2

Settings 탭 클릭

리포 상단 탭에서 Settings를 클릭합니다.

3

Pages 메뉴

왼쪽 사이드바에서 Pages를 클릭합니다.

4

Custom domain 입력

Custom domain 입력란에 구매한 도메인을 입력합니다. 예: myname.kr
Save 버튼을 클릭하면 GitHub가 DNS 검증을 시작합니다.

# GitHub Pages Custom Domain 설정 경로
https://github.com/[계정명]/[리포명]/settings/pages

# y-life.kr 경우
https://github.com/yeon-life/yeon-life.github.io/settings/pages
Chapter 6

HTTPS 인증서 적용 (보안 자물쇠)

커스텀 도메인을 등록하면 GitHub Pages가 자동으로 무료 SSL 인증서(Let's Encrypt)를 발급합니다. 주소창에 자물쇠 아이콘이 표시됩니다.

  • DNS 전파 완료 후 자동 발급 (수 분~수 시간 소요)
  • GitHub Pages Settings → Enforce HTTPS 체크박스를 활성화합니다
  • 이후 http://로 접속해도 자동으로 https://로 리다이렉트됩니다
완료 기준 — 브라우저에서 https://내도메인.kr로 접속했을 때 자물쇠가 표시되면 모든 설정이 완료된 것입니다. 🎉
Chapter 7

자주 묻는 질문 (FAQ)

도메인 연결 후 사이트가 안 열려요. 얼마나 기다려야 하나요?
DNS 전파에 최대 24~48시간이 소요됩니다. 보통 수 분에서 수 시간 내에 완료되지만, 길게는 이틀까지 걸릴 수 있습니다. DNS 전파 확인 사이트(dnschecker.org)에서 진행 상황을 확인할 수 있습니다.
.kr과 .com 중 어떤 것이 좋을까요?
한국 대상 서비스라면 .kr이 더 친근합니다. 가격도 비슷하고 인지도도 높습니다. 글로벌 대상이거나 앞으로 확장 계획이 있다면 .com을 추천합니다. 두 개 모두 구매해서 하나로 연결하는 방법도 있습니다.
가비아 말고 다른 곳에서 구매해도 되나요?
네. 카페24, GoDaddy, 후이즈 등 어디서 구매해도 DNS 레코드 설정 방식은 동일합니다. 이 가이드의 DNS 설정 방법(A 레코드, CNAME 레코드)을 각 서비스의 DNS 관리 페이지에 동일하게 입력하면 됩니다.
도메인 연결 후에도 y-life.kr에 계속 접속할 수 있나요?
y-life.kr 메인 사이트는 별개이므로 계속 접속 가능합니다. 내 커스텀 도메인은 빌더로 만든 내 페이지에만 연결됩니다.
도메인 갱신을 안 하면 어떻게 되나요?
만료 후 일정 기간(보통 30일)이 지나면 도메인이 해제되고 다른 사람이 구매할 수 있습니다. 가비아에서 자동 갱신 설정을 켜두면 매년 자동으로 연장됩니다.
이미 다른 사람이 쓰는 도메인과 비슷한 이름은 쓸 수 없나요?
이미 등록된 도메인은 구매할 수 없습니다. 비슷한 이름이어도 확장자(.kr, .com, .net 등)가 다르면 등록할 수 있습니다. 단, 유명 브랜드나 상표권이 있는 이름은 법적 문제가 생길 수 있으니 조심하세요.
